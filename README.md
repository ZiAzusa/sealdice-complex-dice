# complex-dice (dev branch)

这是 `dev` 分支上的版本说明。

当前分支中的 [complex-dice.dev.js](C:\Users\ziazu\Desktop\sealdice-complex-dice\complex-dice.dev.js) 是实验线实现。`dev` 分支不再保留 `main` 分支中的旧文件布局，而是只保留实验版脚本和对应文档。这个分支的目标是提供一个受限脚本解释器，用于验证更完整的语言特性是否适合后续继续演进。

当前扩展名仍为 `complex-dice-v2-dev`，以便和 `main` 分支中的稳定版本保持隔离。

## 用法

```text
.cd <表达式或脚本>
```

支持：

- 单行表达式
- 多语句脚本
- 换行分隔语句
- `;` 分隔语句
- block 内纯换行分句

多语句脚本的结果为最后一个已执行语句的值。

## 值类型

实验版当前支持四类值：

- 数字
- 字符串
- 布尔值
- 数组

布尔值是第一类公民：

- `true`
- `false`

不再依赖 `0 / 1` 模拟布尔值，但数值在条件判断中仍按“非零为真”处理。

## 变量与赋值

### 局部变量

使用 `auto` 声明局部变量：

```text
.cd auto atk = dice("1d20")
auto total = atk + 5
total
```

变量名支持所有非保留字符的 Unicode 字符。以下字符保留为语法用途，不能出现在变量名中：

```text
空白、换行、" ( ) , ; + - * / % ! > < = ? : & | [ ] { } .
```

例如：

- `攻击`
- `总伤害`
- `变量①`
- `判定结果`
- `$HP`

都可以作为变量名。

### 变量读取顺序

读取变量时按以下顺序查找：

1. 当前作用域链中的局部变量
2. 海豹变量原名
3. 自动补 `$` 后的海豹变量
4. 若仍不存在则报错

例如读取 `HP` 时，会依次尝试：

1. 局部变量 `HP`
2. 海豹变量 `HP`
3. 海豹变量 `$HP`

### 赋值规则

实验版支持普通赋值：

```text
.cd auto 攻击 = dice("1d20")
攻击 = 攻击 + 2
攻击
```

赋值时规则如下：

1. 如果变量已存在于局部作用域链，则修改已有局部变量
2. 否则尝试写入海豹变量原名
3. 若失败且原名不以 `$` 开头，则再尝试写入 `$变量名`
4. 写入后立即回读校验；若写入没生效或变量不可改，则报错

### 显式海豹变量访问

实验版内置：

- `get("变量名")`
- `set("变量名", 值)`

它们与普通变量读写使用同一套海豹变量 fallback 规则，但可以在函数中显式使用。

示例：

```text
.cd set("$g测试方法", "return 1")
.cd get("$g测试方法")
```

## 运算符

### 优先级

| 优先级 | 运算符 | 说明 |
|--------|--------|------|
| 1 | `=` | 赋值 |
| 2 | `? :` | 三元条件 |
| 3 | `\|\|` | 逻辑或 |
| 4 | `&&` | 逻辑与 |
| 5 | `==` `!=` | 相等比较 |
| 6 | `>` `<` `>=` `<=` | 大小比较 |
| 7 | `+` `-` | 加减、字符串拼接 |
| 8 | `*` `/` `//` `%` | 乘、除、整除、取模 |
| 9 | `-` `!` | 一元负号、逻辑非 |
| 10 | `fn()` `a[i]` `a.x` | 调用、下标、成员 |

### 说明

- `+`：若任一操作数为字符串，则执行字符串拼接
- `//`：向下取整的整除
- 比较结果为原生布尔值 `true / false`
- `&&` / `||` 返回布尔值
- 条件判断仍采用“数字非零、字符串非空、数组非空为真”

## 控制流

实验版支持：

- `if / else`
- `while`
- `for (init; condition; update)`
- `break`
- `continue`
- 三元表达式 `? :`

### 语句与表达式双重身份

以下结构既可以作为语句，也可以作为可求值结构放进表达式里：

- `if`
- `while`
- `for`
- `{ ... }` block

它们作为表达式时，结果为最后一个已执行语句的值。

例如：

```text
.cd (if (true) { 1 } else { 0 }) + 2
```

结果为 `3`。

### 限制

- `break` 只能在循环内使用
- `continue` 只能在循环内使用

## 函数

### 定义与调用

实验版支持用户自定义函数：

```text
.cd function add(a, b) {
  return a + b
}
add(2, 3)
```

支持：

- 参数传递
- `return`
- 递归

### 作用域

函数调用会创建独立作用域。

函数内默认不会隐式读写海豹变量。  
如果函数内需要访问海豹变量，请显式使用 `get()` / `set()`。

例如：

```text
.cd function heal(x) {
  return set("HP", get("HP") + x)
}
heal(5)
```

### 限制

- `return` 只能在函数内使用
- 函数参数数量受限制
- 函数调用深度受限制

## block 与作用域

实验版当前采用 block 作用域：

- 顶层脚本有全局作用域
- 每个 `{ ... }` block 都会新建一个作用域
- `for (...) { ... }` 的初始化和循环体在同一个额外作用域中执行
- 函数调用会新建函数作用域

示例：

```text
.cd {
  auto x = 1
}
x
```

上面的 `x` 在 block 外不可见。

block 也可以作为表达式使用：

```text
.cd ({ auto x = 1; x + 2 }) * 3
```

## 数组

### 基础能力

支持：

- 数组字面量：`[]`
- 下标访问：`a[0]`
- 下标写入：`a[0] = 3`
- `a.length`
- `a.push(x)`
- `a.pop()`

### 语义

- 数组按引用传参
- 函数内修改数组会影响外部同一个数组对象
- `length` 只读

示例：

```text
.cd function pushOne(arr) {
  arr.push(1)
  return arr.length
}

pushOne(a)

```

## 字符串、注释与动态执行

### 单行注释

支持 `#` 单行注释，直到该行结束：

```text
.cd auto x = 1  # 这里是注释
x + 2
```

### 字符串

#### 普通字符串

普通双引号字符串 `"..."`：

- 只能单行
- 支持转义
- 直接跨行会报错

```text
.cd "单行字符串"
```

#### 三引号多行字符串

支持 Python 风格三引号字符串 `"""..."""`：

- 可以跨行
- 中间出现普通 `"` 不需要转义
- 换行会被保留为字符串内容

```text
.cd """第一行
第二行
第三行"""
```

### eval

`eval(code)` 会把字符串再次作为实验版脚本执行。

它的行为是：

- 复用当前解释器
- 共享当前作用域链
- 共享已定义的函数和局部变量
- 继续受同一套步数、循环、递归、数组、字符串限制约束
- 会自动兼容“外层字符串解析后，内层普通字符串中的 `\n` 已变成真实换行”的常见嵌套脚本场景

这意味着你可以把代码存进海豹变量后再执行：

```text
.cd set("$g测试方法", """function fib(n) {
  if (n <= 1) {
    return n
  }
  return fib(n - 1) + fib(n - 2)
}
fib(6)""")
```

然后直接：

```text
.cd eval(get("$g测试方法"))
```


## 资源与安全限制

实验版不是无限制脚本环境。当前内置硬限制如下：

- 表达式长度：`512`
- AST 深度：`256`
- 语句数量：`64`
- 单作用域局部变量数量：`64`
- 执行步数：`10000`
- 循环总次数：`1000`
- 数组长度：`256`
- 函数调用深度：`64`
- 单个函数参数数量：`16`
- 字符串长度：`4096`
- 最终输出总长度：`512`

这些限制用于防止：

- 死循环
- 递归爆炸
- 超大数组
- 超长字符串导致过重 GC 和内存压力
- 动态执行绕过停机机制
- 输出内容无限增长

输出长度超出限制时，不报错，而是只保留最后 `512` 个字符。

## 错误处理

实验版会尽量返回具体错误信息，常见包括：

- 语法错误
- 单行字符串直接跨行
- 三引号字符串未闭合
- 未定义变量
- 未定义函数
- 除数不能为零
- 只读变量写入
- 群变量权限不足
- 执行步数过多
- 循环次数过多
- 调用层数过深

### print 与最终结果

`print(x)` 会额外输出一行文本，但自身没有返回值，不会替代最后的表达式结果输出。

例如：

```text
.cd print("第一行")
print("第二行")
3 + 4
```

会在“得到了结果”后依次输出：

```text
第一行
第二行
7
```

如果最后一行本身就是 `print(...)`，则不会因为返回值而重复输出一次。

## 当前语义选择

- block 建作用域
- 数组按引用传参
- 函数内部禁用隐式海豹变量 fallback
- 显式 `get/set` 仍允许海豹变量 fallback
- 控制流和 block 可以作为表达式使用
- 普通双引号字符串单行，三引号字符串多行
