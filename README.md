# 复杂骰子表达式求值器 (complex-dice)

为海豹骰 (SealDice) 添加 `.cd` 命令，支持 C 风格表达式求值，允许在表达式中调用原生骰子功能。

版本: 1.4.0 | 作者: shimakaze

## 说明

本插件的部分代码由 AI 辅助生成，并经过人工整合、修改与测试。
在使用、二次开发或审计时，仍建议按普通手写代码标准进行检查，不应默认其实现完全正确。

## 用法

```
.cd <表达式>
```

支持单行表达式，也支持使用换行或 `;` 分隔的多语句表达式。

### 变量、赋值与多行

可使用 `auto` 声明临时局部变量，变量仅在当前这一次 `.cd` 求值中有效：

```text
.cd auto atk = dice("1d20")
auto total = atk + 5
total > 10 ? "命中" : "未命中"
```

多语句表达式的结果为最后一条语句的值。

变量名支持所有非保留字符的 Unicode 字符。以下字符保留为语法用途，不能出现在变量名中：

```text
空白、换行、" ( ) , ; + - * / % ! > < = ? : & |
```

例如：`攻击`、`总伤害`、`变量①`、`判定结果`、`$HP` 都可以作为变量名。

除 `auto` 声明外，也支持通用赋值：

```text
.cd auto 攻击 = dice("1d20")
攻击 = 攻击 + 2
攻击
```

读取变量时，解释器按以下顺序查找：

1. `auto` 定义的局部变量
2. 原名海豹变量
3. 自动补 `$` 后的海豹变量
4. 若仍不存在，则报错

例如读取 `HP` 时，会依次尝试：局部变量 `HP`、海豹变量 `HP`、海豹变量 `$HP`。

赋值变量时，规则如下：

1. 若同名 `auto` 局部变量已存在，则修改该局部变量
2. 否则尝试写入同名海豹变量
3. 若失败且原名没有 `$` 前缀，则再尝试写入 `$变量名`
4. 每次写入后都会立即回读校验；若海豹拒绝修改、类型不支持或写入未生效，则报错

注意：

- 海豹数值变量按 `int` 处理，因此非整数数值不能写入海豹变量
- 字符串可以写入海豹字符串变量
- 内置函数名不能作为赋值目标

### 运算符（按优先级从低到高）

| 优先级 | 运算符 | 说明 |
|--------|--------|------|
| 1 | `? :` | 三元条件 |
| 2 | `\|\|` | 逻辑或 |
| 3 | `&&` | 逻辑与 |
| 4 | `==` `!=` | 相等比较 |
| 5 | `>` `<` `>=` `<=` | 大小比较 |
| 6 | `+` `-` | 加减、字符串拼接 |
| 7 | `*` `/` `//` `%` | 乘、除、整除、取模 |
| 8 | `-` `!` | 一元负号、逻辑非 |
| 9 | `fn()` | 函数调用 |

- 比较和逻辑运算结果为 `1`（真）或 `0`（假）
- `+` 运算符：若任一操作数为字符串，则执行字符串拼接
- `//` 为整除（向下取整）：`5 // 2 = 2`，`-5 // 2 = -3`
- 字符串字面量仅能作为三元运算结果或参与 `+` 拼接，不能用于其他算术运算

### 内置函数

| 函数 | 说明 | 示例 |
|------|------|------|
| `dice(expr)` | 调用原生骰子，返回数值 | `dice("1d20")` `dice("2d6+1")` |
| `max(a, b, ...)` | 取最大值 | `max(1, 5, 3)` → `5` |
| `min(a, b, ...)` | 取最小值 | `min(1, 5, 3)` → `1` |
| `floor(x)` | 向下取整 | `floor(3.14)` → `3` |
| `ceil(x)` | 向上取整 | `ceil(3.14)` → `4` |
| `round(x)` | 四舍五入 | `round(3.14)` → `3` |
| `abs(x)` | 绝对值 | `abs(-5)` → `5` |
| `str(x)` | 数字转字符串 | `str(42)` → `"42"` |
| `choose(a, b, ...)` | 自动掷 1dN 并随机选一 | `choose("吃晚饭", "不吃晚饭")` |
| `wchoose(a, w1, b, w2, ...)` | 按权重随机选一 | `wchoose("普通命中", 80, "暴击", 15, "大失败", 5)` |

`dice()` 支持标准的骰子表达式（如 `1d20`、`2d6+1`、`d100`），也支持在代骰场景下按被代理人的上下文进行求值。

### 示例

```
.cd 1 + 2 * 3
→ 由于 1 + 2 * 3, 用户名 得到了结果 7

.cd dice("1d20") + 5
→ 由于 dice("1d20") + 5, 用户名 得到了结果 17

.cd dice("1d20") > 10 ? "命中" : "未命中"
→ 由于 dice("1d20") > 10 ? "命中" : "未命中", 用户名 得到了结果 命中

.cd str(floor(dice("1d100") / 2)) + "%"
→ 由于 str(floor(dice("1d100") / 2)) + "%", 用户名 得到了结果 23%

.cd max(dice("2d6"), dice("2d6")) + 3
→ 由于 max(dice("2d6"), dice("2d6")) + 3, 用户名 得到了结果 11

.cd 5 // 2
→ 由于 5 // 2, 用户名 得到了结果 2

.cd choose("吃晚饭", "不吃晚饭", "点外卖")
→ 自动掷 1d3，返回对应选项

.cd dice("1d20") > 10 ? choose("普通命中", "暴击!") : "未命中"
→ 先判 d20 结果，命中后掷 1d2 选效果

.cd wchoose("普通命中", 80, "暴击", 15, "大失败", 5)
→ 自动掷 1d100，按权重返回对应选项

.cd auto first = dice("1d20"); auto total = first + 5; total
→ 先保存一次 d20 结果，再在后续语句中复用

.cd auto 伤害 = dice("1d6")
伤害 = 伤害 + 2
伤害
→ 修改局部变量并返回新值

.cd HP = HP + 1
→ 若局部没有 `HP`，则尝试写入海豹变量 `HP`，失败后再尝试 `$HP`

.cd 判定结果 = "成功"
→ 将字符串写入海豹变量 `判定结果` 或 `$判定结果`
```

## 中文标点

表达式中的中文标点会在字符串字面量之外自动转换为对应英文标点，无需手动切换输入法：

| 中文 | 英文 | 说明 |
|------|------|------|
| `（` `）` | `(` `)` | 括号 |
| `“` `”` | `"` | 中文引号，可作为字符串定界符 |
| `，` | `,` | 逗号 |
| `：` | `:` | 冒号 |
| `？` | `?` | 问号 |
| `！` | `!` | 感叹号 |
| `＋` `－` `＊` `／` `％` | `+` `-` `*` `/` `%` | 全角运算符 |
| `＞` `＜` `＝` | `>` `<` `=` | 全角比较符 |

说明：

- 中文引号 `“”` 可作为字符串字面量的定界符使用
- 字符串字面量内部的内容会保持原样，不会被中文标点归一化

例如：

```
.cd dice（"1d20"） ＞ 10 ？ "命中" ： "未命中"
→ 完全等价于 .cd dice("1d20") > 10 ? "命中" : "未命中"
```

## 限制

- 表达式长度最大为 `512` 字符
- 表达式嵌套深度最大为 `256` 层
- 语句数量最大为 `64` 条
- 临时变量数量最大为 `64` 个
- 海豹变量写入成功后会立即回读校验

## 错误处理

表达式或掷骰调用出错时，会尽量返回具体错误信息，例如：

- 语法错误
- 字符串未闭合
- 未定义的函数
- 除数不能为零
- 掷骰表达式格式错误

## 技术说明

- 纯手写递归下降解析器，不使用 `eval`
- 运算符优先级与 C 语言一致
- 比较/逻辑运算仿 C：非零即真，结果为 `1`/`0`
- `dice()` 通过 `seal.format` 桥接原生骰子引擎
- 支持 `auto` 临时变量和多语句表达式，返回最后一条语句的值
- 支持 Unicode 变量名、通用赋值，以及“局部变量 -> 海豹变量 -> `$`海豹变量”的变量解析顺序
- 包含表达式长度、嵌套深度、语句数量与变量数量限制
